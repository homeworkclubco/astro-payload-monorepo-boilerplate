The spacing system provides a consistent set of CSS custom properties for margins, padding, gaps, and other spacing needs. It uses a modular scale to ensure visual harmony across all components and layouts.

## Spacing Scale

The spacing system uses a 13-step modular scale, from micro spacing to massive spacing. Each step provides both a base value and an optional larger value for flexibility.

| Token         | Base Value | Large Value | Use Case                        |
| ------------- | ---------- | ----------- | ------------------------------- |
| `--space-4xs` | 2px        | 4px         | Micro spacing, hairline borders |
| `--space-3xs` | 4px        | 6px         | Tiny gaps, tight spacing        |
| `--space-2xs` | 6px        | 8px         | Very small spacing              |
| `--space-xs`  | 8px        | 12px        | Small gaps, compact layouts     |
| `--space-sm`  | 12px       | 16px        | Standard small spacing          |
| `--space-md`  | 16px       | 24px        | Default medium spacing          |
| `--space-lg`  | 24px       | 32px        | Standard large spacing          |
| `--space-xl`  | 32px       | 48px        | Extra large spacing             |
| `--space-2xl` | 48px       | 64px        | Very large spacing              |
| `--space-3xl` | 64px       | 96px        | Huge spacing                    |
| `--space-4xl` | 96px       | 128px       | Massive spacing                 |
| `--space-5xl` | 128px      | 192px       | Extra massive spacing           |
| `--space-6xl` | 192px      | 256px       | Giant spacing                   |

## Usage Patterns

### Component Spacing

```css
/* Card component */
.card {
  padding: var(--space-lg);
  margin-bottom: var(--space-md);
}

.card-header {
  margin-bottom: var(--space-sm);
}

/* Button component */
.button {
  padding: var(--space-sm) var(--space-md);
}

.button-large {
  padding: var(--space-md) var(--space-lg);
}

/* Form component */
.form-group {
  margin-bottom: var(--space-md);
}

.form-input {
  padding: var(--space-sm);
}
```

### Layout Spacing

```css
/* Section spacing */
.section {
  padding: var(--space-2xl) 0;
}

/* Container spacing */
.container {
  padding: 0 var(--space-lg);
}

/* Grid spacing */
.grid {
  gap: var(--space-md);
}
```

```css
  padding: var(--space-sm);
}
```

```css
  .responsive-padding {

}

@media (min-width: 1024px) {

    margin: var(--space-sm) 0;
  }
}

  }
}
```

## Best Practices

### Spacing Hierarchy

- **Micro spacing (4xs-2xs)**: Use for borders, hairlines, and very tight interfaces
- **Small spacing (xs-sm)**: Use for related elements within components
- **Medium spacing (md-lg)**: Use for component padding and standard gaps
- **Large spacing (xl-2xl)**: Use for section spacing and major layout divisions
- **Massive spacing (3xl-6xl)**: Use for page-level spacing and hero sections

### Consistency Rules

- **Use the scale**: Always use spacing tokens instead of arbitrary values
- **Be consistent**: Use the same spacing value for similar elements throughout the design
- **Consider context**: Adjust spacing based on content density and user needs
- **Test responsiveness**: Ensure spacing works well across different screen sizes

### Common Patterns

```css
/* Component internal spacing */
.component {
  padding: var(--space-md);
  gap: var(--space-sm);
}

/* Component external spacing */
.component + .component {
  margin-top: var(--space-lg);
}

/* Section spacing */
.section {
  padding: var(--space-xl) 0;
}

.section + .section {
  margin-top: var(--space-2xl);
}
```

## Integration

The spacing tokens are automatically available in all components that import the UI package:

```astro
---
// Component using spacing tokens
---
<style>
  .my-component {
    padding: var(--space-md);
    margin-bottom: var(--space-lg);
    gap: var(--space-sm);
  }
</style>
```

Use spacing tokens consistently to maintain visual harmony and ensure your design system remains scalable and maintainable.
