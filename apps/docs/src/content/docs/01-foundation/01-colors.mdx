We use a two-layer color system:

1.  **Primitives:** The raw palette (Neutral 50-950). We use OKLCH for perceptually uniform lightness steps.
2.  **Semantics:** The logical application (Background, Border, Accent). **Always use semantic tokens in your components.**

## Semantic Tokens

These tokens automatically adapt when the user switches between Light and Dark mode.

### Backgrounds

| Token            | Preview                                                                                                                | Description                             |
| :--------------- | :--------------------------------------------------------------------------------------------------------------------- | :-------------------------------------- |
| `--color-bg`     | <div style="background: var(--color-bg); width: 100%; height: 2rem; border: 1px solid var(--color-border);"></div>     | Default page background.                |
| `--color-bg-alt` | <div style="background: var(--color-bg-alt); width: 100%; height: 2rem; border: 1px solid var(--color-border);"></div> | Secondary background (sidebars, cards). |

### Foreground (Text)

| Token                          | Preview                                                                              | Description                          |
| :----------------------------- | :----------------------------------------------------------------------------------- | :----------------------------------- |
| `--color-foreground`           | <span style="color: var(--color-foreground); font-weight: bold;">Aa</span>           | Primary headings and body text.      |
| `--color-foreground-secondary` | <span style="color: var(--color-foreground-secondary); font-weight: bold;">Aa</span> | Subtitles, captions, and muted text. |

### Borders

| Token                   | Preview                                                                         | Description                        |
| :---------------------- | :------------------------------------------------------------------------------ | :--------------------------------- |
| `--color-border`        | <div style="border: 1px solid var(--color-border); height: 1rem;"></div>        | Subtle dividers and card borders.  |
| `--color-border-strong` | <div style="border: 2px solid var(--color-border-strong); height: 1rem;"></div> | High-contrast boundaries (inputs). |

### Accents (Brand)

## Primitive Palette (Neutrals)

These are the raw ingredients defined in `tokens.css`. Only use these directly if you are creating a new semantic alias.

## Usage Example

```css
.card {
  /* ✅ Good: Adapts to Dark Mode */
  background: var(--color-bg-alt);
  border: 1px solid var(--color-border);
  color: var(--color-foreground);
}

.card-bad {
  /* ❌ Bad: Will be blindingly bright in Dark Mode */
  background: var(--color-neutral-100);
  color: black;
}
```

## Dark Mode Support

The system handles dark mode via the `data-color-scheme="dark"` attribute on the `<html>` or `<body>` tag.

Semantically, the mappings invert logic:

- Light Mode: `--color-bg` maps to Neutral-50 (White-ish).
- Dark Mode: `--color-bg` maps to Black (Dark).

```astro
// Force dark mode on a specific section
<section data-color-scheme="dark">
  <Card>I am dark even on light mode!</Card>
</section>
```
